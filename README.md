# B&S Influence Dashboard

Social influence performance dashboard for Barker & Stonehouse — paid influencer campaigns (Meta), organic/gifted posts, and GA4 UTM-attributed traffic.

Live at: `https://dlawrence-bands.github.io/bs-influence/`

---

## Repo structure

```
bs-influence/
├── index.html                        # Dashboard (GitHub Pages entry point)
├── data/
│   ├── organic_posts.json            # Influencer organic/gifted posts (manually maintained)
│   ├── paid_ads.json                 # Paid campaign daily data (from Influencer-Report.xlsx)
│   └── ga4_influencer.json           # Auto-generated by Python script — DO NOT edit manually
├── scripts/
│   └── fetch_ga4_influencer.py       # BigQuery → ga4_influencer.json
└── .github/workflows/
    └── refresh_data.yml              # GitHub Action: runs every Monday 07:00 UTC
```

---

## Data sources

### 1. Organic / gifted posts — `data/organic_posts.json`
Influencer posts tagged with B&S. Populated from the Meta Instagram content export.
Edit manually to add new posts or update the `product` field (or use the editable field in the dashboard UI — it saves to localStorage).

Key fields: `id, creator, platform, handle, tier, date, content_type, product, reach, impressions, likes, comments, saves, permalink, gifted, paid`

### 2. Paid campaigns — `data/paid_ads.json`
Generated from the `Influencer-Report.xlsx` export (Meta Ads Manager, ad-level daily data).
To update: re-export from Meta, then run the processing script or paste into the dashboard.

### 3. GA4 / BigQuery — `data/ga4_influencer.json`
Auto-generated. Filters on `utm_medium=influencer`. Run locally:

```bash
pip install google-cloud-bigquery pandas pyarrow db-dtypes
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json  # Windows: set GOOGLE_APPLICATION_CREDENTIALS=C:\path\to\key.json
python scripts/fetch_ga4_influencer.py
```

---

## GitHub Actions setup (automated weekly refresh)

1. Create a GCP service account with **BigQuery Data Viewer** + **BigQuery Job User** roles
2. Download the JSON key
3. Go to **repo → Settings → Secrets and variables → Actions**
4. Add secret named `GCP_SERVICE_ACCOUNT_KEY` → paste the full JSON key contents
5. The action runs every Monday at 07:00 UTC and commits updated `data/ga4_influencer.json`

---

## GitHub Pages setup

1. Push repo to `dlawrence-bands/bs-influence`
2. Settings → Pages → Source: **Deploy from branch** → `main` / `/ (root)`
3. Dashboard live at `https://dlawrence-bands.github.io/bs-influence/`

## Updating paid data

Re-export `Influencer-Report.xlsx` from Meta Ads Manager (same format), then run:

```bash
python scripts/process_paid.py
```

Or drag and drop in the dashboard UI — note the UI version only updates your local view, not the repo data.
